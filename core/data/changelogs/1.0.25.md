# 🎉 What's New in v1.0.25

We are excited to announce version 1.0.25, featuring critical bug fixes and workflow improvements to enhance your database management experience.

## 🐛 Bug Fixes

- **Variable Handling:** Fixed an issue where variables set to `NULL` or `0` were not being processed correctly.
- **Non-Public Schemas:** Resolved `DELETE` and `UPDATE` operation failures on tables located outside the default "public" schema.

## 🔧 Improvements

### Quick Query

- **Safe Mode:** Introduced a new "Safe Mode" to prevent accidental data loss. This feature adds a safeguard when executing `UPDATE` or `DELETE` statements.
- **Error Logging:** Enhanced error log details within the Log Panel for better debugging and traceability.
- **Function Editor:** Added support for editing functions, allowing users to modify and save function definitions.

### ERD (Entity Relationship Diagram)

- **Visual Relationships:** Added support for ERD visualization, allowing users to explore table relationships directly within the table view.

### Raw Query Editor

- **Syntax Highlighting:** Added specific highlighting for SQL functions to improve code readability.
- **Error Tab:** Introduced a dedicated "Error" tab in the results pane for clearer error reporting.
- **Smart Autocomplete:** Improved the intelligence of schema suggestions (IntelliSense) for faster and more accurate query writing.

## ⚠️ Breaking Changes

- **Raw query editor:** Not support can parser -- PARAMETERS: [...] in comment of query, because this feature is only for internal use when execute query from UI, not for user write query in editor, so this not support for now, we will consider to support in future if have demand from user
