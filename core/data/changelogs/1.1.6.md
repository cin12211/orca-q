# 🎉 What's New in v1.1.6

Version 1.1.6 introduces direct data editing in Raw Query, a refined data grid wrapper, keyboard shortcut updates, and robust system reset capabilities.

## ✨ New Features

### Raw Query Data Mutation

- **Editable Raw Queries:** You can now edit cell values directly in Raw Query results!
- **Relation Previews:** View and browse related tables directly from Raw Query tab sheets.

### Workbench & Code Editor

- **Reload Routine Definition:** Added a dedicated **Reload** button (with **`⌘R`** / **`Ctrl+R`** / **`F5`** keyboard shortcuts) to refresh the latest view/function definition from the database.
- **Unsaved Modifications Warning:** Modifying a routine and trying to reload now prompts a premium confirmation dialog to prevent accidental work loss.

---

## 🔧 Improvements

### Sizing & Grid Refactor

- **Global Sizing Configuration:** Setup component sizing globally for unified UI/UX consistency across all windows and panels.
- **Table Grid Refactoring:** Extensively refactored and optimized the AG Grid wrapper to improve performance, rendering speed, and clean up context menus.
- **Storybook Updates:** Aligned Storybook stories with the latest component structure and sizing options.

---

## 🐞 Bug Fixes

### Editor & Shortcuts

- **Routine Save Shortcut:** Resolved the issue where using `⌘S`/`Ctrl+S` failed to save routine/function definitions or incorrectly displayed empty previews. The phím tắt now triggers a direct save, matching the tooltip behavior.
- **Save Event Error:** Fixed a classic event parameter bug where clicking "Save to database" inside the preview dialog passed a `MouseEvent` instead of the SQL string, resulting in database execution errors.

### System Settings

- **Reset User Data Redirect:** Fixed a critical redirect issue in the Electron app when performing "Reset all data". The application now reloads in-place instead of incorrectly redirecting the window to the local root filesystem (`file:///`).
- **Reset Foreign Key Violations:** Reimplemented Electron user data deletion within a single database transaction. This ensures SQLite connection-scoped foreign key checks (`PRAGMA foreign_keys = OFF`) are respected properly by Knex's pool, ensuring a 100x faster and constraint-error-free reset.
