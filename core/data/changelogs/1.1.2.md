# 🎉 What's New in v1.1.2

Version 1.1.2 brings support for new database types, a brand-new Redis module, and several UI/UX improvements to make your workflow smoother.

## ✨ New Features

### Database Support

- 🚀 **New SQLite Providers:** Added support for **Cloudflare D1** and **Turso**.
- <img src="https://api.iconify.design/devicon:redis.svg" width="16" style="display:inline-block;vertical-align:middle;margin-right:6px"/>**Redis Support:** You can now connect to Redis instances to browse keys, manage TTLs, and use Pub/Sub messaging.
- **Enhanced Schema Loading:** Better performance and reliability when working with databases that have many schemas.

### Workbench & UI

- **Quick SQL Access:** Added a dedicated "SQL" button to quickly jump into a scratchpad or open existing scripts.
- **New Tab Menu:** A new "+" button lets you easily create files, browse schemas, or open instance insights.
- **Environment Visibility:** Your connection's environment (e.g., Production, Staging) is now clearly visible in the connection selector and status bar.
- **Table Customization:** You can now set a global preference for how `NULL` values are sorted in your data tables.

## 🔧 Improvements

- **Quick Query Preview:** Added a "View Changes" button so you can double-check your updates before saving them.
- **Better Redis Browsing:** Improved the key tree navigation and flattened list views for easier searching.
- **Polished UI:** Updated the header styles across the app and added more sizing options for input fields.

## 🐞 Bug Fixes

- **Data Integrity:** Fixed an issue where the value `0` was sometimes cleared out when saving data.
- **Query Editor:** Resolved errors when running queries that contained specific commented-out lines.
- **Tab Management:** Tabs will now automatically close if the underlying file is deleted from your workspace.
- **Visual Fixes:** Fixed a UI bug in the JSON editor where text would become hard to read when selected.
